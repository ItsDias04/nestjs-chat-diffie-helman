# 🔧 Исправление проблемы с SQLMap

## Проблема
SQLMap установлен, но запускается как `python3 sqlmap`, а не просто `sqlmap`.

## Решение

Скрипты были обновлены для автоматического определения правильной команды SQLMap.

## Как запустить сейчас

### Вариант 1: Запустить диагностику (рекомендуется)

```bash
cd ~/Рабочий\ стол/sqlmap/security-testing
python3 find_sqlmap.py
```

Этот скрипт:
- Найдет SQLMap в вашей системе
- Протестирует его работу
- Сохранит правильную команду

### Вариант 2: Запустить основной скрипт напрямую

```bash
cd ~/Рабочий\ стол/sqlmap/security-testing
sudo python3 sqlmap_automation.py
```

Теперь скрипт автоматически определит:
- `sqlmap` (если в PATH)
- `python3 sqlmap` (ваш случай)
- `python3 ~/Рабочий\ стол/sqlmap/sqlmap.py` (если установлен локально)

### Вариант 3: Установить SQLMap через apt (альтернатива)

```bash
sudo apt-get update
sudo apt-get install sqlmap
```

## Что было исправлено

### В файле `sqlmap_automation.py`:

1. **Умное определение команды SQLMap** - проверяет несколько вариантов:
   - `sqlmap` (команда в PATH)
   - `python3 sqlmap` (ваш случай)
   - Локальные установки в домашней директории

2. **Динамическое использование команды** - автоматически подставляет правильную команду

### В файле `quick_test.py`:

Аналогичные улучшения для быстрого тестирования

## Если проблемы остались

Укажите явный путь к SQLMap в config.env:

```bash
# Добавьте в config.env
SQLMAP_CMD=python3 sqlmap
```

Или:

```bash
# Для локальной установки
SQLMAP_CMD=python3 /home/idekireuko/Рабочий\ стол/sqlmap/sqlmap.py
```
